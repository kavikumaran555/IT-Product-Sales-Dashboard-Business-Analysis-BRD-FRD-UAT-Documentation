# IT Product Sales Dashboard – Business Analysis Documentation (BRD, FRD, UAT)

## Project Overview

This project is a Business Analysis case study focused on designing documentation for a Sales Dashboard system for an IT product company.

It includes complete requirement analysis and system design documents such as Business Requirements Document (BRD), Functional Requirements Document (FRD), User Acceptance Testing (UAT) test cases, and a system data flow diagram.

The objective is to demonstrate Business Analyst skills in requirement gathering, documentation, process design, and validation for a sales analytics system.

---

## Objective

To define and document a Sales Dashboard system that provides visibility into the performance of IT products such as laptops, modems, and processing chips by tracking key business metrics including:

- Sales quantity
- Revenue
- Profit
- Product returns
- Loss due to returns
- Period-wise performance comparison
- Product performance ranking
- Customer buying patterns
